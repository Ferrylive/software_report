# 口腔健康评估报告 - 在线部署指南

## 🚀 快速部署（推荐：Netlify Drop）

### 步骤：

1. **访问 Netlify Drop**
   - 打开浏览器，访问：https://app.netlify.com/drop

2. **上传文件**
   - 将整个 `version3` 文件夹拖拽到页面上
   - 或者点击选择文件夹，选择 `version3` 文件夹

3. **获取链接**
   - 等待上传完成（通常几秒钟）
   - 你会得到一个类似这样的URL：`https://random-name-123.netlify.app`
   - 这个链接可以在手机上打开测试

4. **手机测试**
   - 在手机上打开这个URL
   - 测试所有功能：
     - 查看报告内容
     - 点击牙齿查看详情
     - 测试滑动和交互功能

### 注意事项：

- ✅ 确保上传整个 `version3` 文件夹（包括所有子文件夹）
- ✅ 确保 `images` 文件夹完整上传
- ✅ 如果图片无法显示，检查浏览器控制台是否有错误
- ✅ 报告会自动使用嵌入的示例数据（如果 `result.json` 无法加载）

---

## 📱 其他部署方式

### GitHub Pages

1. 在GitHub创建新仓库
2. 上传所有文件到仓库
3. 在 Settings > Pages 中启用 GitHub Pages
4. 访问 `https://你的用户名.github.io/仓库名/`

### Vercel

1. 访问 https://vercel.com
2. 使用GitHub登录
3. 导入仓库或上传文件夹
4. 自动部署完成

## 📂 文件结构

```
version3/
├── report.html          # 主报告文件
├── result.json          # 报告数据（可选，有嵌入的示例数据）
├── images/              # 图片文件夹
│   ├── Processed image/ # 牙齿PNG图片
│   └── ...              # 其他图片
└── README.md            # 本文件
```

---

## 🔧 技术说明

- 报告会自动尝试加载 `result.json`
- 如果无法加载，会使用嵌入的示例数据（`EMBEDDED_DATA`）
- 所有图片路径都是相对路径，确保文件夹结构正确
- 支持移动端响应式设计

---

## ❓ 常见问题

**Q: 图片无法显示？**
A: 检查 `images` 文件夹是否完整上传，路径是否正确。

**Q: 数据无法加载？**
A: 报告会自动使用嵌入的示例数据，不影响测试。

**Q: 手机端显示异常？**
A: 清除浏览器缓存，或使用无痕模式打开。
