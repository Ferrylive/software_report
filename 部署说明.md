# 在线部署说明

## 方法一：Netlify Drop（最简单，推荐）

1. 访问 https://app.netlify.com/drop
2. 将整个 `version3` 文件夹拖拽到页面上
3. 等待上传完成，会得到一个临时URL（例如：`https://random-name-123.netlify.app`）
4. 在手机上打开这个URL即可测试

**优点**：无需注册，无需配置，拖拽即可使用

---

## 方法二：GitHub Pages（详细步骤）

### 第一步：准备GitHub账号

1. 如果没有GitHub账号，访问 https://github.com 注册
2. 如果已有账号，直接登录

### 第二步：创建新仓库

1. 登录GitHub后，点击右上角的 **"+"** 按钮
2. 选择 **"New repository"**（新建仓库）
3. 填写仓库信息：
   - **Repository name**（仓库名称）：输入 `dental-report`（或你喜欢的名称）
   - **Description**（描述）：可选，例如 "口腔健康评估报告"
   - **Visibility**（可见性）：选择 **Public**（公开）或 **Private**（私有）
     - ⚠️ **注意**：GitHub Pages 免费版只支持 Public 仓库
   - **不要**勾选 "Add a README file"（我们会上传自己的文件）
4. 点击 **"Create repository"**（创建仓库）

### 第三步：上传文件到仓库

#### 方法A：使用GitHub网页上传（最简单）

1. 创建仓库后，会看到一个快速设置页面
2. 点击 **"uploading an existing file"**（上传现有文件）链接
3. 在文件上传页面：
   - 将 `version3` 文件夹中的所有文件**拖拽**到页面上
   - 或者点击 **"choose your files"** 选择文件
   - ⚠️ **重要**：需要上传所有文件，包括：
     - `report.html`
     - `result.json`（可选，有嵌入数据）
     - `images` 文件夹（整个文件夹及其所有内容）
     - 其他所有文件
4. 滚动到页面底部，填写提交信息：
   - **Commit message**：例如 "Initial commit - 上传报告文件"
5. 点击 **"Commit changes"**（提交更改）

#### 方法B：使用Git命令行（适合有Git经验的用户）

```bash
# 1. 进入version3文件夹
cd d:\D0_com\workplaceplus\C1_software_report\version3

# 2. 初始化Git仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交文件
git commit -m "Initial commit - 上传报告文件"

# 5. 添加远程仓库（替换YOUR_USERNAME为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/dental-report.git

# 6. 推送到GitHub
git branch -M main
git push -u origin main
```

### 第四步：启用GitHub Pages

1. 在仓库页面，点击右上角的 **"Settings"**（设置）标签
2. 在左侧菜单中，找到并点击 **"Pages"**（页面）
3. 在 **"Source"**（源）部分：
   - 选择 **"Deploy from a branch"**（从分支部署）
   - **Branch**（分支）：选择 `main` 或 `master`
   - **Folder**（文件夹）：选择 `/ (root)`（根目录）
4. 点击 **"Save"**（保存）

### 第五步：访问你的网站

1. 等待1-2分钟，让GitHub Pages完成部署
2. 在 **"Pages"** 设置页面，你会看到：
   ```
   Your site is live at https://YOUR_USERNAME.github.io/dental-report/
   ```
3. 点击这个链接，或在浏览器中访问：
   ```
   https://YOUR_USERNAME.github.io/dental-report/report.html
   ```
   ⚠️ **注意**：如果 `report.html` 是主页，可能需要访问完整路径

### 第六步：设置默认页面（可选）

如果希望直接访问 `https://YOUR_USERNAME.github.io/dental-report/` 就能看到报告：

1. 在仓库中，点击 **"Add file"** > **"Create new file"**
2. 文件名输入：`index.html`
3. 在文件内容中，输入重定向代码：
   ```html
   <!DOCTYPE html>
   <html>
   <head>
       <meta http-equiv="refresh" content="0; url=report.html">
   </head>
   <body>
       <p>正在跳转...</p>
   </body>
   </html>
   ```
4. 点击 **"Commit new file"**

### 手机测试

1. 在手机上打开浏览器
2. 访问：`https://YOUR_USERNAME.github.io/dental-report/report.html`
3. 测试所有功能

### 常见问题

**Q: 上传后网站显示404？**
A: 
- 检查文件是否都上传成功
- 确保访问的是完整路径：`/dental-report/report.html`
- 等待几分钟让GitHub Pages完成部署

**Q: 图片无法显示？**
A: 
- 检查 `images` 文件夹是否完整上传
- 确保图片路径是相对路径（代码中已经是相对路径）

**Q: 如何更新文件？**
A: 
- 在仓库页面点击文件，然后点击编辑按钮
- 或者重新上传文件覆盖旧文件
- 提交更改后，GitHub Pages会自动更新（可能需要几分钟）

**优点**：
- ✅ 永久免费
- ✅ 可以自定义域名
- ✅ 版本控制（可以回退到之前的版本）
- ✅ 支持HTTPS

## 方法三：Vercel

1. 访问 https://vercel.com
2. 使用GitHub账号登录
3. 点击 "New Project"
4. 导入你的GitHub仓库（或直接上传文件夹）
5. 部署完成后会得到一个URL

**优点**：自动部署，支持自定义域名

## 注意事项

- 确保 `images` 文件夹及其所有子文件夹都正确上传
- 如果图片无法显示，检查图片路径是否正确
